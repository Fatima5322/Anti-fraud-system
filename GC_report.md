# 1. Анализ результатов тестирования сервиса DaData

## 1.1 Прямое геокодирование

В ходе тестирования было выполнено 100 запросов на прямое геокодирование.

Среднее время обработки одного запроса составило 155,22 мс.

Для оценки качества прямого геокодирования использовалось поле qc_geo, которое возвращает DaData.

В ходе анализа отчетов по результатам запросов установлено, что для 28 запросов сервис вернул в ответе пустой массив suggestions. Данные ответы не содержали никакой информации и не участвовали в дальнейшем анализе точности. 

Для оставшихся 72 запросов результат оказался следующим:

|Значение qc_geo|Интерпретация|Кол-во запросов|
|-|-|-|
|0|Точные координаты объекта|32|
|2|Координаты ближайшего дома|40|

Данные результаты показывают, что в случаях, когда сервис находил нужный адрес, он возвращал координаты высокой точности. Чаще всего результат соответствовал уровню qc_geo = 2 (определение координат ближайшего дома).

## 1.2 Обратное геокодирование 

В ходе тестирования было выполнено 100 запросов на обратное геокодирование. 

Среднее время обработки одного запроса составило 134,82 мс.

По сравнению с прямым геокодированием обратное выполнялось быстрее.

Для оценки качества обратного геокодирования использовалось поле qc_geo.

Во всех выполненных запросах сервис успешно определил объект с максимальной точностью (qc_geo = 0).

Для определения уровня детализации использовалось поле fias_level, которое возвращает DaData.

Результат следующий:

|Уровень детализации|Кол-во запросов|
|-|-|
|8 (дом)|100|

Во всех случаях сервис определил адрес до уровня дома, что является максимально возможной степенью детализации. Это говорит о высокой точности результатов обратного геокодирования DaData.

# 2. Анализ результатов тестирования сервиса Яндекс Геокодер

## 2.1 Прямое геокодирование

В ходе тестирования сервиса Яндекс Геокодер было выполнено 100 запросов на прямое геокодирование.

Среднее время обработки одного запроса составило 367,52 мс.

По сравнению с DaData среднее время оказалось выше, однако полученное значение так же приемлемо для большинства задач.

Для оценки качества результатов использовалось поле precision, которое возвращает Яндекс Геокодер.

Результат следующий:

|Значение precision|Интерпретация|Кол-во запросов|
|-|-|-|
|exact|Точное совпадение адреса|84|
|number|Совпадение до номера дома|14|
|near|Ближайший объект|2|

Большинство запросов (84%) были обработаны с максимальной точностью (exact). В остальных случаях сервис определял объект до уровня номера дома (number) либо до ближайшего объекта (near).

## 2.1 Обратное геокодирование

В ходе тестирования сервиса Яндекс Геокодер было выполнено 100 запросов на обратное геокодирование.

Среднее время обработки одного запроса составило 166,36 мс. 

Обратное геокодирование выполнялось быстрее прямого, однако время ответа оказалось несколько выше, чем у DaData.

Для обратного геокодирования Яндекс Геокодер также возвращает поле precision.

Результат следующий:

|Значение precision|Интерпретация|Кол-во запросов|
|-|-|-|
|other|Определен объект без высокой точности адреса|85|
|отсутствует|Адрес не найден|15|

В большинстве случаев сервис возвращал значение other, которое говорит о том, что был найден некоторый географический объект, однако без высокой адресной точности. Для 15 запросов подходящий объект найден не был. 

Поэтому максимальный уровень детализации оценивался по типу найденного объекта (kind).

Результат следующий:

|Тип объекта (kind)|Кол-во запросов|
|-|-|
|province|34|
|area|26|
|hydro|15|
|other|8|
|locality|1|
|district|1|
|Адрес не найден|15|

По результатам видно, что в большинстве случаев сервис воозвращал объекты уровня региона или административной территории, а не полноценные почтовые адреса.

# 3. Анализ результатов тестирования сервиса GigaChat

## 3.1 Прямое геокодирование

В ходе тестирования было выполнено 100 запросов на прямое геокодирование.

Среднее время обработки одного запроса составило 792,61 мс, что значительно превышает аналогичный показатель двух предыдущих сервисов. Это связано с тем, что GigaChat представляет собой языковую модель, выполняющую генерацию текста, а не специализированный сервис геокодирования.

В отличие от специализированных сервисов геокодирования, GigaChat не предоставляет собственных показателей качества (как qc_geo и precision). Поэтому оценка проводилась на основе соответствия ответов требованиям, указанным в промпте.

Результат следующий:

|Метрика|Результат|
|-|-|
|Ответов, содержащих JSON|100|
|Ответов, содержащих дополнительный текст после JSON|100|
|Ответов с техническими комментариями (округление, нормализация адреса и тд)|70|
|Ответов с предупреждением о приблизительности координат|7|
|Ответов с рекомендацией использовать специализированные API|23|

Анализ показал, что во всех случаях модель возвращала JSON требуемой структуры, однако ни один ответ не соответствовал заданной инструкции полностью, поскольку после JSON добавлялись пояснения, комментарии и рекомендации.

## 3.2 Обратное геокодирование

В ходе тестирования было выполнено 100 запросов на обратное геокодирование.

Среднее время обработки одного запроса составило 1958,96 мс, что является наибольшим показателем среди всех исследуемых сервисов.

Результат следующий:

|Метрика|Результат|
|-|-|
|Ответов, содержащих JSON|100|
|Ответов, содержащих дополнительный текст после JSON|94|
|Ответов с рекомендацией использовать специализированные API|75|
|Ответов, содержащих сообщение о невозможности определить адрес либо отказ выполнять задачу|59|

Анализ показал, что при обратном геокодировании модель значительно чаще сообщала о собственных ограничениях. В большинстве подобных случаев GigaChat рекомендовал использовать специализированные сервисы геокодирования либо сообщал, что определить точный адрес без обращения к внешним картографическим сервисам невозможно. Кроме того, вместо определения фактического адреса модель нередко приводила пример корректного JSON или пример возможного ответа, который не соответствовал переданным координатам.

# 4. Рекомендации

На основании проведенного тестирования и анализа результатов можно сформулировать следующие рекомендации по выбору сервиса геокодирования:

- DaData рекомендуется использовать в системах, где приоритетными являются высокая скорость обработки запросов и получение детализированной структурированной информации об адресе.
- Яндекс Геокодер рекомендуется использовать в случаях, когда важны максимальная полнота обработки запросов, устойчивость к различным форматам записи адресов и высокая вероятность получения результата даже для сложных или неоднозначных запросов.
- GigaChat целесообразно применять только в качестве вспомогательного инструмента для обработки естественного языка или анализа текстовых данных. Использование модели в качестве самостоятельного сервиса геокодирования не рекомендуется из-за отсутствия специализированных механизмов определения координат и нестабильного следования заданному формату ответа.
